# 半年英语突破计划

> 目标：6个月内达到英文口语、听力、阅读、沟通基本无障碍
> 起点：大学毕业多年，有一定英语基础和英文技术阅读经验
> 每日投入：3~5 小时

---

## 目标定义

| 维度 | 半年后目标 | 不是目标 |
|------|-----------|---------|
| CEFR | B2 → C1 | 母语级 C2 |
| 雅思 | 6.5~7.5 | 满分 |
| 工作沟通 | 基本无障碍 | 完美口音 |
| 技术阅读 | 几乎无压力 | 文学级表达 |
| YouTube | 80%+ 听懂 | 100% |
| 口语 | 连续聊 30~60 分钟 | 辩论级 |

---

## 你的优势

1. **不是零基础** — 有语法和词汇底子，不是从零开始
2. **技术背景天然适合** — 长期接触英文技术文档，逻辑思维强
3. **AI 时代红利** — 随时有 AI 陪练、纠错、模拟场景，这是过去没有的
4. **技术领域兴趣** — AI/编程/科技类英文内容你本身就爱看，更容易坚持

---

## 核心原则（贯穿全程）

```
英语能力 = 大量可理解输入 + 高频输出 + 听力自动化 + 英语思维速度
```

**必须抛弃：**
- 学校英语学习法（背单词表、做语法题、逐词翻译）
- "学英语" 的心态

**必须建立：**
- "用英语生活" 的心态
- 英语 = 操作系统，不是学科
- 沉浸式输入 + 高频输出

---

## 第一阶段：重建系统（第 1~2 个月）

> 最关键阶段。大脑在重建语言系统，会很痛苦，但必须坚持。

### 每日安排（约 4 小时）

#### 1. 精听训练（60 分钟）

这是核心中的核心。

**材料要求：** 难度略高于当前水平 + 有字幕 + 真人表达

**推荐材料：**

| 类型 | 推荐 |
|------|------|
| 技术向 | YouTube 技术频道、Microsoft 官方、OpenAI 访谈、开发者播客 |
| 通用英语 | TED 演讲、YouTube vlog |
| 剧集 | Friends、Modern Family |

**精听方法（不是"看"）：**

```
1. 听一句
2. 暂停
3. 跟读（模仿语音语调）
4. 不会的查
5. 重复
6. 一小时大概只学 3~5 分钟内容 — 这是正常的
```

**关键：** 质量 > 数量。宁可精听 5 分钟，不要泛听 1 小时。

---

#### 2. 影子跟读 Shadowing（45 分钟）

口语提升最快的方法之一。

**方法：** 听到什么，立刻同步模仿（不是背诵）

**重点模仿：**
- 节奏
- 连读
- 发音
- 语气

**材料：** TED、美剧、YouTube 博主

**进阶：** 从慢速开始，逐步跟上正常语速

---

#### 3. 大量阅读（60 分钟）

建立"英语直接理解能力"——不要翻译成中文。

**材料：**

| 优先级 | 材料 |
|--------|------|
| 最高 | 技术博客、GitHub issue、Stack Overflow |
| 高 | Reddit（r/programming, r/artificial）、Hacker News |
| 中 | Medium、英文新闻 |
| 低 | 英文小说（后期加入） |

**阅读原则：**

```
❌ 每个单词都查
✅ 猜意思 + 快速阅读 + 只查影响理解的关键词
```

**目标：** 速度、语感、结构感

---

#### 4. 口语输出（60 分钟）

中国人最缺的一环。必须每天开口。

**方式：**
- ChatGPT Voice / Claude 对话（纯英文）
- 英语陪练 App（如 Speak、ELSA）
- 自言自语（见下方）

**每天至少 1 小时纯英文交流**

**话题轮换：**
- 周一：技术（解释你写的代码、讨论架构）
- 周二：日常（今天做了什么、计划）
- 周三：AI/科技新闻
- 周四：工作（项目、bug、方案讨论）
- 周五：观点（对某事的看法）
- 周六：自由话题
- 周日：复盘本周学到了什么

**注意：** 不要怕卡顿。提升口语的关键不是"不犯错"，而是高频输出。

---

### 第一阶段里程碑

- [ ] 能精听 5 分钟 TED 演讲，理解 80%+
- [ ] 能跟读跟上正常语速的 60%+
- [ ] 能阅读 Medium 技术文章，不逐词翻译
- [ ] 能用英文连续说 2~3 分钟（即使有错误）
- [ ] 手机系统已改为英文

## 第二阶段：自动化（第 3~4 个月）

> 目标：听力自动化、建立英文思维、能持续交流

### 关键转变：全面英语化

**必须做到：**

- [ ] 手机系统改英文
- [ ] 浏览器默认英文
- [ ] YouTube 只看英文内容
- [ ] Google 搜索优先英文
- [ ] 技术资料只看英文原版
- [ ] AI 对话只用英文
- [ ] 想法尝试用英文表达

### 每日安排（约 4 小时）

#### 1. 听力进阶（60 分钟）

从精听过渡到"半泛听"：

```
70% 精听（新内容，逐句跟读）
30% 泛听（通勤/做饭时听播客、YouTube）
```

**新增材料：**
- 英文播客（技术类：Software Engineering Daily、Lex Fridman）
- YouTube 长视频（20~40 分钟）

#### 2. 口语深化（60 分钟）

从"能说"到"能持续说"：

**训练重点：**
- 连续说 5~10 分钟不卡
- 能解释复杂概念
- 能表达观点和论证

**新增训练：**
- 用英文解释你写的代码给"别人听"
- 用英文做技术方案讲解（对着屏幕自言自语）
- 用英文描述遇到的 bug 和解决过程

#### 3. 阅读加速（60 分钟）

**目标：** 阅读速度接近中文的一半

**训练：**
- 限时阅读（计时）
- 读完用英文写 2~3 句总结
- 开始接触英文技术书籍（如 Clean Code、Designing Data-Intensive Applications）

#### 4. 英文思维训练（60 分钟）

**日常自言自语（极强训练）：**

```
I'm making coffee now.
I need to finish this API today.
This bug is annoying.
I think the issue is in the middleware.
Let me check the logs first.
```

**为什么有效：** 你在建立英语直连思维，而不是"中文→翻译→英文"。

#### 2. 高级输入（60 分钟）

| 类型 | 推荐 |

## 词汇策略

### 不要传统背单词

**正确方式：** 在高频输入中自然习得

一个词在 YouTube、Reddit、博客、AI 对话中连续出现 20 次，你自然记住。

### 需要专门背的

高频口语表达和连接词，比 GRE 单词重要得多：

```
actually, basically, kind of, pretty much, to be honest
I mean, you know, the thing is, at the end of the day
in terms of, as a result, on the other hand, having said that
```

### 工具推荐

- **Anki** — 间隔重复，只收录你在真实场景中遇到的生词
- **欧路词典** — 查词 + 生词本
- **Readlang** — 浏览器插件，阅读时点词即查并自动收集

## 常见问题

### Q: 前两个月很痛苦怎么办？
正常。大脑在重建语言系统。表现会像：听不懂、跟不上、说不出来、大脑很累。第 3 个月会突然开始"流动"。

### Q: 每天真的需要 4 小时吗？
3 小时是底线，4 小时理想。如果某天只有 1 小时，优先精听 + 口语输出。

### Q: 要不要学音标？
第一周花 2~3 天过一遍英语音标（美式），然后在跟读中自然纠正。

### Q: 要不要上语法课？
不需要系统学语法。在阅读和听力中自然强化。遇到不确定的语法点，问 AI 即可。

### Q: 口音重要吗？
清晰 > 标准。目标是让别人听懂，不是变成播音员。

### Q: 坚持不下来怎么办？
- 找一个学习伙伴（线上/线下）
- 在 Discord/社区打卡
- 记录每天的学习时间
- 允许自己有"低能量日"，但不要连续停 2 天

## 最后的话

**不要把英语当学科。把英语变成你的操作系统。**

- 看 AI 用英语
- 学技术用英语
- 搜索用英语
- 娱乐用英语
- 思考用英语

当英语成为"工具"而不是"学科"时，你会进步得极快。

**关键心法：**
```
不是"学英语"
而是"用英语生活"
```

半年后，你会感谢今天的自己。
